# TZ Mate v0.1.0 Prerelease

## Overview

This is a developer/testing prerelease of TZ Mate, a native macOS menu bar app and widget for checking client local time, converting time zones, and planning international calls.

This prerelease is intended for source builds, QA, and early feedback. It is not yet a fully polished signed and notarized public binary release.

## What Is Included

- Native macOS menu bar app
- Compact SwiftUI popover
- Contacts tab
- Time converter tab
- Settings tab
- WidgetKit widget extension
- Local country, phone code, city, and time zone data
- Local-first storage
- Shared storage between the main app and widget through App Groups

## Features

- Save client contacts with country, city, phone code, note, and IANA time zone
- See current client local time
- See practical time status labels
- Mark favorites for widget display
- Search contacts
- Copy local time summary
- Look up countries by phone code
- Handle shared phone codes such as `+1` and `+7`
- Handle large countries with multiple time zones
- Convert time between your time zone and a client time zone
- Copy converter result
- Choose system, light, or dark theme
- Choose 12-hour or 24-hour time format
- Show favorite contacts in a macOS widget
- Use Settings to check for updates through Sparkle once releases are configured

## Installation

For this prerelease, build from source using Xcode.

```sh
xcodebuild -list
xcodebuild -scheme TZMate -configuration Debug build CODE_SIGNING_ALLOWED=NO
xcodebuild -scheme TZMateWidgetExtension -configuration Debug build CODE_SIGNING_ALLOWED=NO
```

A GitHub binary release will come later.

## Update Checking Note

TZ Mate includes a Sparkle-based update-checking foundation for future GitHub Releases distribution.

Production update delivery requires a signed and notarized app, a Sparkle-signed release archive, and an HTTPS `appcast.xml`. Until the final appcast is published, the update controls are available for prerelease testing but production updates are not active.

## Development Build Note

Unsigned local builds are useful for development and QA. They are not recommended for broad public users because macOS Gatekeeper may show warnings.

Signed local builds require selecting a Development Team in Xcode and enabling App Groups for both targets:

```text
group.com.remdee.tzmate
```

## Signing And Notarization Note

For public distribution outside the Mac App Store, TZ Mate should be signed with a Developer ID certificate and notarized by Apple.

The Mac App Store is not planned for the initial version.

## Known Limitations

- The app icon is a temporary placeholder.
- Screenshots are not yet captured.
- Launch at Login requires a properly signed local build.
- Widget editing is not implemented; contacts are edited in the main app.
- Sparkle updates require a published appcast and signed release archive.
- Public signed and notarized binaries are not available yet.

## Privacy

- No account required
- No backend
- No analytics
- No tracking
- Contacts and settings are stored locally on the user's Mac
- The widget reads local shared storage only
